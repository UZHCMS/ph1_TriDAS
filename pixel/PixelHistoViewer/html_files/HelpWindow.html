    • ~~ Getting Started:
    The Pixel Histogram Viewer environment is designed to be similar to the desktops of common Operating Systems. At the bottom of the browser there is the Console. The Console allows for quick navigation between windows. Clicking a window's icon will bring that window to the front.

    The two most important windows are the File Navigator and Detector Navigator windows. As their names suggest, the File Navigator allows the user to browse the available data in a file/directory view while the Detector Navigator allows for graphical browsing.

    To proceed, you will want to get familiar with the     Quick Keys and read through the Basics of Browsing section.

    ~~ Basics of Browsing:
    The File Navigator is where all browsing should begin. In the Files Pane (the left pane) a list of available files are shown. If "< no files >" is displayed then try clicking the Refresh button; if it persists then there are no files currently available on the network.

    The Content Pane (the right pane) displays all the directories and data present in the current File. If "< no content - refresh file list >" is displayed try clicking the Refresh button and then the desired File in the left pane again; if it persists then the file currently has no content.

    Browsing through the directories, the user may find Histograms and Summaries. Clicking on a Histogram will open up a new window displaying the chosen Histogram. Clicking on a Summary will display a list of Fields from within the Summary. Then clicking on a Field will display and re-color the ROC's of the Detector Navigator window based on the Summary and Field that was clicked.

    Within the Detector Navigator, the user can interact with each and every ROC of the CMS Pixel Detector. By left clicking on a portion of the detector, the user can Zoom in an area of interest. When zoomed, the ROC's are interactive - by positioning the mouse over a particular ROC and left clicking, the histograms in the current File related to that ROC will be displayed. By right clicking, the user returns to the full detector view.

    ~~ Configuration Cookies:
    Clicking the Save Cookies icon at the upper-right corner of the desktop will save the state of the Pixel Histogram Viewer. It saves the size and position of all windows as well as the Canvas Controls parameters. The browser will remember the user settings every time the user returns to the viewer (the cookies expire in one year).

    To return to the default configuration, click the Clear Cookies icon on the desktop.

    ~~ The Filter String:
    The Filter String is used to manipulate the display of Summaries within the Detector Navigator. This section describes the syntax and effect of the Filter String. Follow these steps to access the Filter:
  • Open the File Navigator.
  • Open a File by clicking on an item in the left pane.
  • Locate in the right pane the folder named "SummaryTrees." If this folder does not exist then there are no summaries available for the current file.
  • Expand the avaliable Summaries and click a Summary Field. A valid Summary Field must always begin with "Field - ".
  • The Detector Navigator should open after a few seconds with the Filter String text field visible in the top right.

    • Syntax Rules:
    There are 3 types of commands for filtering the summary data: Sigma(S), Min/Max(M), and Bool(B). Sigma commands use a scaled value of the standard deviation to set the min and max of the color scale around the average. Min/Max commands set the min and max of the color scale directly. Bool commands color the ROC's green or red based on passing or failing a condition. The following conventions were adopted here:
  • _ => Any amount of white space including none
  • [p#] => A floating point parameter, where # is some integer
  • Avg => Average of all values in the Summary
  • Sigma => Standard devialtion of all values in the Summary
  • {csv} => Comma-separated variables within curly brackets indicate the set of valid values
  • O(*) => Indicates * is an optional field
  • v => Summary value associated with a roc

    • Sigma Command Details:
    Syntax: _ S _ [p1] _

    Example: "S 10000"
    Result: scale min = Avg - 10000*Sigma, scale max = Avg + 10000*Sigma

    Min/Max Command Details:
    Syntax: _ M _ {<,>} _ [p1] _ O({<,>} _ [p2] _ )
    (<= and >= have same effect as < and >)

    Example 1: " M < 1.87"
    Result: scale max = 1.87
    Example 2: "M > -1.5"
    Result: scale min = -1.5
    Example 3: "M >-1.5 < 10 "
    Result: scale min = -1.5, scale max = 10

    Bool Command Details:
    Syntax: _ B _ {<,<=,>,>=,=} _ [p1] _ O({&,|} _ {<,<=,>,>=} _ [p2] _ )

    Example 1: "B = 1.87"
    Result: green if v = 1.87, else red
    Example 2: " B <5.5 "
    Result: green if v < 5.5, else red
    Example 3: "B < 5 | >=7.2"
    Result: green if v < 5 or v >= 7.2, else red
    Example 4: " B > 5 & <= 10 "
    Result: green if 5 < v <= 10, else red

    ~~ Speed Tips:
    The Pixel Histogram Viewer has been tested and is known to work with Firefox 2/3 and Safari browsers. If possible, use Safari, because the viewer may run significantly faster. If using Firefox, it is recommended to disable Firebug for speed considerations.
